# Ex Origine v0.2-v0.6 Closeout

This milestone extends the v0.1 core runtime into the first observable simulation loop.

## Milestone Map

- v0.2: Observable world snapshots.
- v0.3: Contextual entity behavior.
- v0.4: Replay inspection.
- v0.5: Player influence modes.
- v0.6: Terminal observer grid.

## Acceptance Questions

- Can the simulation export one readable world snapshot per tick?
- Can an entity make decisions from local world context instead of a fixed action only?
- Can replay data summarize and inspect runtime causality?
- Can the player influence the world without direct entity control?
- Can a person watch the world state evolve in a terminal view?

## Runtime Outputs

- `runtime/replays/latest_world.jsonl`: JSON-lines `WorldSnapshot` stream.
- `runtime/replays/latest_events.replay`: event replay stream.

## Verification

Run from the repository root:

```sh
./scripts/verify_v0_6.sh
```

or on Windows PowerShell:

```powershell
.\scripts\verify_v0_6.ps1
```

The verifier builds the app, runs a 10-tick default loop, runs a 100-tick observable loop with `--mode attract`, checks missing config failure, and runs unit, MVB integration, v0.6 observer integration, and legacy smoke tests.

## Manual Observer Check

After building with the verifier:

```powershell
.\build\v0_6_verify\ex_origine.exe --ticks 20 --mode attract --observe
```

Expected result:

- The terminal prints `[OBSERVE]` headers each tick.
- The grid shows resource intensity using text glyphs.
- Entity position appears as `E`.
- The entity's last action and memory signal are visible below the grid.

## v0.7 Handoff

The next natural milestone is a small persistent observer interface:

- load a saved `latest_world.jsonl` snapshot stream;
- scrub tick-by-tick;
- inspect entity memory/action history;
- compare influence modes across runs;
- keep deterministic replay as the causality backbone.
